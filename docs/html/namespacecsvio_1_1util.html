function to escape characters in csv fields according to RFC 4180

Source: https://tools.ietf.org/html/rfc4180

If discoverd that the string doesn't have any characters which need to be escaped, the string will not be enclosed in quotes

Also handles alternative delimiters and allows forced escapes which will append leading and trailing double quotes

Generally O(n) avg time and space complexity

Parameters
datastring to escape
delimused to check if a delimiter is in the field. if so, it should be escaped
force_escapewrap the string in quotes even if not needed
Returns
escaped csv field if necessary, otherwise the string
